Instructions for Replit — Improve AgentBricks Scaffolding

Paste the following as a prompt or system instructions to the Replit scaffolding agent.

1 · agent.py — No external config file

Do not generate a config.yaml (or any YAML/JSON config file) that is read by agent.py at module level. Instead, declare all configuration as module-level Python constants:

# ── Agent configuration ────────────────────────────
_LLM_ENDPOINT     = "databricks-meta-llama-3-3-70b-instruct"  # verify endpoint exists in workspace
_MAX_TOKENS       = 4096
_TEMPERATURE      = 0.1
_MAX_ITERATIONS   = 20
_COMPLETION_TOKEN = "TASK_COMPLETE"
_EXPERIMENT_NAME       = "/Shared/<agent_name>_experiment"
_REGISTERED_MODEL_NAME = "<catalog>.<schema>.<agent_name>"
_SERVING_ENDPOINT_NAME = "<endpoint_name>"

LLM endpoint name: always use databricks-meta-llama-3-3-70b-instruct as the default. The 3-1 variant does not exist in most workspaces and causes a 404 ENDPOINT_NOT_FOUND at inference time.

Reason for inlining: MLflow code-based logging re-imports agent.py inside the serving container where no sidecar files are present. A missing config.yaml raises FileNotFoundError and kills the endpoint.

2 · agent.py — Serving auth via resources= (critical)

This is the most important fix. Always declare resources=[DatabricksServingEndpoint(...)] in log_model. This is what causes Databricks Model Serving to inject a token for the LLM endpoint into the container. Without it, ChatDatabricks has no credentials and every request fails with ValueError: cannot configure default credentials.

from mlflow.models.resources import DatabricksServingEndpoint

mlflow.pyfunc.log_model(
    name="agent",                    # use name=, not artifact_path= (deprecated)
    python_model=__file__,
    code_paths=["tools", "pyproject.toml"],
    signature=signature,
    input_example=input_example,
    pip_requirements=pip_requirements,
    resources=[DatabricksServingEndpoint(endpoint_name=_LLM_ENDPOINT)],
    registered_model_name=_REGISTERED_MODEL_NAME,
)

Note: ChatDatabricks() at module level does NOT need explicit host/api_key arguments. The resources= declaration handles credential injection automatically at serve time.

3 · agent.py — Databricks credentials for deploy only

Read credentials from ~/.databrickscfg for use by WorkspaceClient in __main__ (deploy time only). Do not pass these to ChatDatabricks — it relies on resources= at serve time.

import configparser, os

# Deploy-time credentials from `databricks configure --token`
_dbcfg = configparser.ConfigParser()
_dbcfg.read(os.path.expanduser("~/.databrickscfg"))
_DB_HOST  = _dbcfg.get("DEFAULT", "host",  fallback=None)
_DB_TOKEN = _dbcfg.get("DEFAULT", "token", fallback=None)

# Module-level LLM — no explicit auth, resources= handles it at serve time
llm = ChatDatabricks(endpoint=_LLM_ENDPOINT, max_tokens=_MAX_TOKENS, temperature=_TEMPERATURE)

# In __main__ only:
w = WorkspaceClient(host=_DB_HOST, token=_DB_TOKEN)

4 · agent.py — __file__ guard for serverless

Always add this guard immediately after imports, before any path construction:

if "__file__" not in globals():
    __file__ = sys._getframe(0).f_code.co_filename

Reason: Databricks serverless spark_python_task executes scripts via exec(compile(...)) in an IPython kernel. __file__ is never injected, so os.path.dirname(__file__) raises NameError without this guard.

5 · agent.py — Explicit ModelSignature (never infer_signature)

Always build the signature manually. Never call infer_signature():

from mlflow.models import ModelSignature
from mlflow.types.schema import Array, ColSpec, DataType, Object, Property, Schema

_chat_msg = Array(Object([
    Property("role",    DataType.string),
    Property("content", DataType.string),
]))
signature = ModelSignature(
    inputs=Schema([ColSpec(type=_chat_msg, name="messages")]),
    outputs=Schema([ColSpec(type=_chat_msg, name="messages")]),
)

input_example = {
    "messages": [{"role": "user", "content": "<a realistic example query>"}]
}

Reason: infer_signature calls the LLM endpoint at registration time. On a first deploy the endpoint does not yet exist, raising a network error and halting deployment. input_example suppresses a UserWarning and enables one-click testing in the Databricks UI.

6 · agent.py — Endpoint creation in __main__

Always include create/update logic for the serving endpoint. The EndpointCoreConfigInput constructor requires name:

existing_endpoints = {ep.name for ep in w.serving_endpoints.list()}
if _SERVING_ENDPOINT_NAME in existing_endpoints:
    w.serving_endpoints.update_config_and_wait(
        name=_SERVING_ENDPOINT_NAME,
        served_entities=[served_entity],
        timeout=timedelta(minutes=30),
    )
else:
    w.serving_endpoints.create_and_wait(
        name=_SERVING_ENDPOINT_NAME,
        config=EndpointCoreConfigInput(
            name=_SERVING_ENDPOINT_NAME,   # required — do not omit
            served_entities=[served_entity],
        ),
        timeout=timedelta(minutes=30),
    )

7 · pyproject.toml — Dependency rules for Databricks serverless

  1. Never pin packages pre-installed on serverless (mlflow, databricks-sdk, pydantic, pyarrow) in [project.dependencies]. Exact == pins cause version-conflict errors during serverless pip install.
  2. Put pre-installed packages in [project.optional-dependencies] dev with loose constraints so local development works.
  3. Only packages not pre-installed on serverless go in [project.dependencies] with loose version ranges.
  4. Place [build-system] at the top of the file.
  5. Add a [tool.hatch.build] exclude list that at minimum excludes .databricks/, dist/, .github/, and .env*.
  6. Do not include pyyaml — there is no config.yaml.
[project.dependencies]
  "databricks-langchain",
  "langchain-core>=0.3,<0.4",

[project.optional-dependencies.dev]
  "mlflow>=2.18.0", "databricks-sdk>=0.36.0",
  "pydantic>=2.0.0,<3.0.0", "pyarrow>=14.0.0",
  "pytest>=7.4.0,<9.0.0"

8 · databricks.yml — Do NOT generate an artifacts: section

Reference the pre-built wheel directly as a local path in spec.dependencies. DAB uploads it automatically:

environments:
  - environment_key: default
    spec:
      client: "2"
      dependencies:
        - ./dist/<package_name>-${var.wheel_version}-py3-none-any.whl

Do not add an artifacts: block with type: whl. DAB's whl artifact builder looks for setup.py and fails on pyproject.toml-only projects.
