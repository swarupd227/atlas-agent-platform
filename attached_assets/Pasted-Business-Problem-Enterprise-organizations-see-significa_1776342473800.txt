Business Problem
Enterprise organizations see significant revenue impact from transaction stalls, abandonments, and declines. Exceptions — unresponsive signer, form abandonment, wrong recipient, technical access issue, recipient-declined — are managed reactively by operations staff manually reviewing the Sender UI dashboard. By the time an intervention is made, a deal may have already slipped.
Demo Narrative & Workflow
Atlas deploys a digital agreement operations pipeline that monitors every active OneSpan Sign transaction in real time, classifies exceptions (including the Recipient Declined exception shown in the walkthrough), orchestrates targeted recovery actions, and surfaces a live operations intelligence view to the business team.
1.	Transaction Health Monitor polls the OneSpan Sign API (Sender UI layer) every 15 minutes; computes a completion risk score per in-flight transaction; flags at-risk transactions before SLA expiry.
2.	Exception Classifier categorizes each at-risk or declined transaction across five exception types: (1) Stalled at signer — unresponsive; (2) Form abandoned mid-completion; (3) Wrong recipient — routing error; (4) Technical block — authentication/access issue; (5) Recipient Declined — signer explicitly rejected the document with a stated reason.
3.	Intervention Orchestrator selects the right recovery action per exception: automated personalized re-send for stalled signers; corrected document workflow for declined transactions (reads decline reason from OneSpan Sign API, determines if the issue is correctable, sends corrected document or escalates to relationship manager); alternative signer routing for wrong recipient; IT support ticket for technical blocks.
4.	Agreement Operations Intelligence Agent aggregates transaction health data from the OneSpan Sign Analytics API into a real-time completion rate dashboard — by business unit, document type, authentication method, and signer channel — surfacing completion trends, decline reason patterns, and ROI impact to business leadership.

  OneSpan Sign Signer UI Event API MCP: Signer UI layer: signing ceremony events, consent accept/decline events, decline reason text, signer authentication outcomes, and completion timestamps
  OneSpan Sign Analytics API MCP: Analytics layer: completion rates, time-in-flight, drop-off stages, authentication success rates, usage trends, and decline rate reporting
  CRM Integration MCP: Salesforce / Dynamics: account priority lookup, relationship manager escalation, and deal stage synchronisation
  IT Helpdesk MCP: ServiceNow / Jira: technical block escalation ticket creation and resolution tracking

Governance & Compliance Policies
  Re-Send Frequency Policy — automated re-sends capped at 2 per transaction within 48 hours to prevent signer fatigue
  Declined Document Correction Policy — corrected document resend requires sender approval when the correction involves substantive document content changes
  Data Minimisation Policy — Atlas reads transaction metadata, status events, and decline reason codes only; agreement document content is not accessible to Atlas agents
