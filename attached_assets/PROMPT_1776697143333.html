🏗️ Mandatory Architecture Patterns

Agent Structure

The prompt enforces one Orchestrator and N specialized agents from your spec, all inheriting from BaseAgent or RetryableAgent.

Orchestrator Responsibilities

ResponsibilityImplementation
Empty query rejection{"status": "error", "error": "Query must not be empty"}
Query normalizationquery = query.strip() before any agent
Collision-safe workflow IDsuuid4().hex — never time.time()
Monitoring isolationAll monitor.log() wrapped in try/except
Low-confidence fallbacksearch_with_fallback retries with broader intent
⚠️ Never use time.time() for workflow IDs. Under concurrent load, millisecond timestamps collide. Always use uuid4().hex.

⚙️ Configuration — Single Source of Truth

All values live in src/config/settings.py. No hardcoded catalog names, endpoints, or paths anywhere else.

@dataclass
class MonitoringConfig:
    catalog: str = "workspace"
    schema:  str = "default"
    table:   str = "<your_table>"

    @property
    def full_table(self) -> str:
        return f"{self.catalog}.{self.schema}.{self.table}"

@dataclass
class ServingConfig:
    endpoint_name:  str = "<your_endpoint>"
    workload_size:  str = "Small"
    scale_to_zero:  bool = True

    @property
    def full_model_name(self) -> str:
        return f"{self.model_name_catalog}.{self.model_name_schema}.{self.model_name}"

🔄 Retry Logic

🚨 Critical: On retry attempts, call execute() directly — never super().run(). Calling super().run() on retries creates duplicate nested MLflow spans that pollute traces. 
for attempt in range(self.max_retries + 1):
    if attempt > 0:
        delay = self.retry_delay * (2 ** (attempt - 1))  # exponential backoff
        time.sleep(delay)

    # First attempt: use BaseAgent.run() to create the MLflow span
    # Retries: call execute() directly to avoid nesting spans
    if attempt == 0:
        result = super().run(input_data)
    else:
        result = self.execute(input_data)

📊 MLflow Tracing

🚨 Always use /Shared/ experiment paths. Service principals cannot resolve /Users/ paths — their username resolves to a UUID, producing a path that does not exist. 
# Required at the top of every entry point and load_context()
mlflow.set_tracking_uri("databricks")
mlflow.set_registry_uri("databricks-uc")

# MUST use /Shared/ — never /Users/<username>/
mlflow.set_experiment("/Shared/<your_experiment_name>")
python
⚠️ Without mlflow.set_tracking_uri("databricks") in load_context(), the serving container defaults to a local SQLite file and throws sqlite3.OperationalError.

🛡️ Guardrails

Three mandatory layers. A failed guardrail stops the pipeline immediately and returns a structured response — it does not log a warning and continue.

LayerLocation in PipelineFile
1 — InputBefore any agent runssrc/guardrails/input_guardrails.py
2 — RoutingAfter triage, before searchsrc/guardrails/routing_guardrails.py
3 — OutputBefore returning to caller, before monitor.log()src/guardrails/output_guardrails.py

1️⃣ Input Guardrails

CheckRuleAction
Empty/whitespacelen(query.strip()) == 0Block
Min length< min_query_lengthBlock
Max length> max_query_lengthBlock
Repetition floodSingle char/word repeated > 20×Block
Prompt injectionMatches pattern list (full phrase, case-insensitive)Block
PIIEmail / phone / card / SSN regexMask → pass sanitized
 
@dataclass
class GuardrailResult:
    passed: bool
    reason: str = ""
    sanitized_query: str = ""

# Orchestrator integration
guardrail_result = self.input_guardrails.check(query)
if not guardrail_result.passed:
    return {"status": "blocked", "reason": guardrail_result.reason, "workflow": {...}}
query = guardrail_result.sanitized_query
python
ℹ️ PII is masked, not blocked. The sanitized query ([EMAIL], [PHONE] etc.) is passed downstream. Never log the original PII-containing query.

2️⃣ Routing Guardrails

CheckRuleAction
Low confidenceconfidence < thresholdForce escalation
Unknown intentIntent not in triage_categoriesForce escalation
Explicit escalationintent == "escalation"Force escalation
Escalation flag conflictneeds_escalation=True at high confidenceEscalation flag wins
⚠️ High confidence does not override an escalation flag. Always check both the guardrail decision AND the triage needs_escalation flag — escalate if either is true.

3️⃣ Output Guardrails

CheckRuleAction
Empty responselen(response.strip()) == 0Replace with safe fallback
Too shortword_count < response_min_wordsAdd "response_too_short" warning
No sourcessources_count == 0 and not escalatedAdd "no_sources" warning
PII in responseRegex match on generated textMask before returning
Low confidenceconfidence < min_output_confidenceDowngrade to escalation
🚨 Output guardrail must run before monitor.log(). The sanitized response — not the raw one — is what gets logged to the Delta table.

📦 Python Wheel Packaging

The wheel must install as <pkg>.* not src.*. Use explicit package_dir:

setup(
    name="<your_package_name>",
    packages=[
        "<pkg>", "<pkg>.agents", "<pkg>.tools",
        "<pkg>.orchestrator", "<pkg>.tracing",
        "<pkg>.config", "<pkg>.guardrails",
    ],
    package_dir={
        "<pkg>":                 "src",
        "<pkg>.agents":        "src/agents",
        "<pkg>.tools":         "src/tools",
        "<pkg>.orchestrator":  "src/orchestrator",
        "<pkg>.tracing":       "src/tracing",
        "<pkg>.config":        "src/config",
        "<pkg>.guardrails":    "src/guardrails",
    },
    entry_points={
        "console_scripts": [
            "main=<pkg>.main:main",
            "register=<pkg>.register_model:register",
        ]
    },
)

🚀 Model Registration & Serving

🚨 Always create a separate serving_model.py logged via code_paths. Never inline the model class in register_model.py — cloudpickle serializes by reference, causing ModuleNotFoundError in the serving container. 
# serving_model.py — loaded via code_paths, not cloudpickle
class YourModel(mlflow.pyfunc.PythonModel):
    def load_context(self, context):
        import mlflow
        # Required — without this, serving defaults to local SQLite
        mlflow.set_tracking_uri("databricks")
        mlflow.set_registry_uri("databricks-uc")

        from <pkg>.config.settings import settings
        from <pkg>.tracing.tracer import initialize_tracer
        initialize_tracer(
            experiment_name=settings.tracing.experiment_name,
            enable_tracing=settings.tracing.enable_tracing,
        )

🗂️ DAB Configuration

🚨 Never use mode: development. It prefixes all resource names with dev_<username>_, which breaks Unity Catalog model registration. 
bundle:
  name: <your_bundle_name>

workspace:
  host: <your_workspace_host>

resources:
  jobs:
    main_job:
      name: "<Your Job Name>"
      environments:
        - environment_key: serverless
          spec:
            client: "2"   # "1"=REPL (unsupported), "2"=Jobs
            dependencies:
              - ./dist/<pkg>-0.1.0-py3-none-any.whl
      tasks:
        - task_key: main_task
          environment_key: serverless
          python_wheel_task:
            package_name: <pkg>
            entry_point: main

  registered_models:
    main_model:
      name: workspace.default.<model_name>
      catalog_name: workspace
      schema_name: default

targets:
  dev:
    workspace:
      root_path: /Workspace/Users/${workspace.current_user.userName}/.bundle/${bundle.name}/dev
