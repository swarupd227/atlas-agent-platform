Frontend Changes
5.1 Pipeline Editor — Composite Stage Type
File: apps/web/src/components/PipelineEditor.tsx (or equivalent)
Changes:
•	Add composite to the stage type picker (alongside agent, approval_gate, parallel_group)
•	When user selects composite, show a Team Selector dropdown (list of agent teams)
•	After team selection, auto-compute and display the execution plan (wave visualization)
New UI Component: CompositeStageConfigurator
// Inline in the Pipeline Editor when stage_type === 'composite'
<CompositeStageConfigurator
  teamId={stage.team_id}
  onTeamSelect={(teamId) => updateStage({ team_id: teamId })}
  executionPlan={executionPlan}     // fetched from API
  stateSchema={stateSchema}         // fetched from API
  errorStrategy={stage.config?.error_strategy || 'fail_fast'}
  onErrorStrategyChange={(strategy) => updateStage({ config: { error_strategy: strategy } })}
/>

5.2 Team Graph Editor — State Key Assignment
File: apps/web/src/components/TeamGraphEditor.tsx (or equivalent)
Changes:
•	Each node in the Team Graph Editor gets a new property panel field: State Key
•	The State Key determines which field in the shared state this node writes to
•	Auto-suggest: default state_key = node_id with hyphens replaced by underscores
•	Show warning if two nodes have the same state_key (conflict)
New property panel fields per node:
┌─────────────────────────────────────┐
│ Node: template-grounding             │
│                                      │
│ Agent:      [MCG-TPL-GROUND-001  ▼]  │
│ State Key:  [template_grounding    ]  │
│ Timeout:    [30000 ms              ]  │
│ Output Schema: [{ ... } Edit      ]  │
│ Fallback:   [{ ... } Edit          ]  │
│ Retry:      [2 attempts, exp backoff] │
│ Node Type:  [● Agent  ○ Team Ref   ]  │
└─────────────────────────────────────┘

New node type option: "Team Reference" — when selected, instead of picking an agent, the user picks another Agent Team. This enables nesting.
